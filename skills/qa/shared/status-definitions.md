# Status Definitions

## Requirements

| Status | Meaning |
|--------|---------|
| **DRAFT** | Newly extracted, not yet validated by user |
| **ACTIVE** | Confirmed requirement (user promoted from DRAFT) |
| **RETIRED** | No longer applicable, kept for traceability |

## Test Cases

| Status | Meaning |
|--------|---------|
| **DRAFT** | Newly derived, not yet validated by user |
| **COVERED** | At least one matching test exists and is mapped |
| **UNCOVERED** | No matching test found |
| **PARTIAL** | Some but not all expected test paths are covered |
| **RETIRED** | No longer applicable, kept for traceability |
